# Шаблоны конфигурации Mihomo для MikroTik

Готовые конфигурационные шаблоны для развёртывания контейнера [Mihomo на маршрутизаторах MikroTik](https://hub.docker.com/r/wiktorbgu/mihomo-mikrotik).

Шаблоны приведены для иллюстративных целей, содержат комментарии и позволяют построить конфигурацию под ваши персональные нужды. Не стесняйтесь их редактировать, убирать лишнее и присылать PR если придумали улучшение или вариант шаблона, который подойдет и остальным пользователям.

## Быстрый старт

### 1. Загрузите файл шаблона

Выберите подходящий вам файл из таблицы ниже и скопируйте его на роутер через **SFTP** чтобы он был расположен в папке:

```
/usb1/docker_configs/mihomo_mikrotik/template/
```

Если к шаблону прилагается .sh файл положите его в папку для **user_sh** скриптов:

```
/usb1/docker_configs/mihomo_mikrotik/user_sh/
```

Путь могут отличаться, если вы меняли настройки по-умолчанию и у вас контейнер расположен в другом хранилище.

### 2. Активируйте шаблон

Установите окружение (ENV) контейнера на загруженный шаблон через параметр (key) CONFIG набора параметров (Mihomo). В команде ниже замените `имя-шаблона.yaml` на выбранный файл (например, `stargate-lite.yaml`):

```bash
/container envs add list=MIHOMO key=CONFIG value="имя-шаблона.yaml"
```

### 3. Перезапустите контейнер

Перезапуск нужен для применения и обновления шаблона, а также, после того как вы его отредактировали, чтобы изменения применились.

```bash
/container stop mihomo_mikrotik
/container start mihomo_mikrotik
```

Имя контейнера (`mihomo_mikrotik`) может отличаться, если вы меняли настройки по-умолчанию.

## Описание шаблонов

Шаблоны содержат избыточный код, который вам может не потребоваться. Если они вам также нужны раскомментируйте их и модифицируйте код согласно [официальному примеру](https://github.com/MetaCubeX/mihomo/blob/Meta/docs/config.yaml). Поскольку это файлы ШАБЛОНОВ конфигурации, а не сами конфигурации, они содержат переменные со знаком `$` и могут не поддерживаться вашим YAML редактором.

### Облегчённый (Lite)

Оптимальный выбор для большинства домашних сетей. Просто, надежно, экономно.

Файл шаблона: [stargate-lite.yaml](stargate-lite.yaml)

**Особенности:**
- 💾 Минимальное потребление памяти (~20 МБ) — работает даже на старых маршрутизаторах
- 🎯 Все необходимое для работы, без лишнего
- ⚡ Быстрая загрузка конфигурации
- 📱 Встроенные переключатели для основных сервисов (YouTube, Telegram, игры, AI)
- 🤖 Поддержка AI сервисов: OpenAI, Google Gemini, Anthropic Claude, Perplexity, Microsoft
- 🌐 Полная поддержка IPv6 и Sniffer (анализ трафика)
- 🔐 AWG группы для WireGuard/AmneziaWG прокси
- 🇷🇺 RU группы для российских прокси (RU_AUTO, RU_MANUAL, RU_FASTEST, RU_FAILOVER)

**Когда использовать:**
- Первый раз настраиваете Mihomo? Выбирайте Lite
- Маршрутизатор устаревающий или с ограниченной памятью? Lite
- Не нужна очень тонкая настройка? Lite — правильный выбор

**Когда НЕ использовать:**
- Нужны правила для CDN провайдеров (Cloudflare, Fastly, Akamai)? → Full
- Нужны GEOIP правила для автоматической маршрутизации по странам? → Full
- Нужны правила для облачных хостингов (AWS, Oracle, Hetzner)? → Full
- Нужны Torrent правила? → Full

### Полный (Stargate)

Для гиков Mihomo с расширенными возможностями и переключателями.

Файл шаблона: [stargate.yaml](stargate.yaml)

**Особенности:**
- 🌍 **ZKEEN геоданные** — автоматическая загрузка актуальных списков IP и доменов по странам (обновление каждые 24 часа)
- 🎮 Избыточно большой набор переключателей для тонкой настройки
- 🛠️ Возможность направлять разные хостинги и сервисы через разные прокси
- 📊 Специальные GEOIP правила для соцсетей, мессенджеров, игр, стриминга
- ☁️ Правила для облачных провайдеров: Amazon, Google, Cloudflare, Fastly, Akamai, CDN77, DigitalOcean, Hetzner, MEGA, Oracle, OVH, Vultr
- 🤖 AI сервисы: OpenAI, Google Gemini, Anthropic Claude, Perplexity, Microsoft Copilot
- 📥 Torrent правила для торрент-клиентов и трекеров
- 🚀 ECH-Refilter для ускорения интернета
- 🇷🇺 Ru-Traffic — отдельная группа для российского трафика по GEOIP
- 💪 Требует больше памяти (~40 МБ+)

**Когда использовать:**
- Разобрались с Lite и хотите большей гибкости?
- Знакомы с настройкой прокси?
- Ваш маршрутизатор мощный и памяти предостаточно?
- Нужна автоматическая маршрутизация по странам через ZKEEN?
- Нужны правила для CDN и облачных хостингов?

**Совет:** Сначала попробуйте Lite, потом переходите на Full если понадобится

### Минимум (Mini)

Для минимальной настройки на очень ограниченном оборудовании **с сервисами**.

Файл шаблона: [stargate-mini.yaml](stargate-mini.yaml)

**Особенности:**
- 🪶 Самое экономичное потребление памяти (~15 МБ)
- 📌 Только самое важное: YouTube, Telegram, WhatsApp, Discord, Cloudflare
- ⚙️ Минимум переключателей — меньше путаницы
- 🐢 Может работать даже на очень древних маршрутизаторах
- ❌ **Без AI сервисов** (нет OpenAI, Gemini, Anthropic и др.)
- ❌ **Без игровых правил** (нет Steam, Twitch)
- ❌ **Без GEOIP/ZKEEN геоданных**
- ❌ **Без IPv6** (отключён для экономии ресурсов)
- ❌ **Без Sniffer** (анализ трафика отключён)

**Когда использовать:**
- Маршрутизатор выпущен 5+ лет назад и памяти всего 32-64 МБ?
- Вам в принципе не нужна тонкая настройка?
- Хотите только основной функционал и ничего больше?
- Не нужны AI сервисы и игры?

**Важно:** Если у вас обычный и современный маршрутизатор — лучше выберите Lite

### Крохотный (Nano)

Для минимальной настройки на очень ограниченном оборудовании **без сервисов**.

Файл шаблона: [stargate-nano.yaml](stargate-nano.yaml)

**Особенности:**
- 🪶 Самое экономичное потребление памяти (~15 МБ)
- ⚙️ Минимум переключателей — только SELECTOR, AUTO, MANUAL, DIRECT
- 🐢 Может работать даже на очень древних маршрутизаторах
- ❌ **Без сервисных групп** (нет Telegram, YouTube, Discord и др.)
- ❌ **Без AI сервисов**
- ❌ **Без GEOIP/ZKEEN геоданных**
- ❌ **Без IPv6** (полностью отключён)
- ❌ **Без Sniffer** (анализ трафика отключён)
- ❌ **Без AWG групп** (нет WireGuard/AmneziaWG поддержки)
- ❌ **Без RU групп** (нет отдельных российских прокси)
- ❌ **Без rule-providers** (нет внешних наборов правил)

**Когда использовать:**
- Маршрутизатор выпущен 5+ лет назад и памяти всего 32-64 МБ?
- Вам в принципе не нужна тонкая настройка?
- Хотите только базовую маршрутизацию без разделения по сервисам?
- Не нужны никакие дополнительные функции?

**Важно:** Если у вас обычный и современный маршрутизатор — лучше выберите Lite

### Цепочки (Chain)

Специальный режим: трафик идет сначала через RU сервер, потом в нужную страну. Полезно если у вас белые списки или нужна иная логика маршрутизации.

Чтобы шаблон работал правильно необходимо в контейнер положить ДВА файла из которых он состоит.

  - Файл шаблона: [stargate-chain.yaml](chain/stargate-chain.yaml)
  - Скрипт: [stargate-chain.sh](chain/user_sh/stargate-chain.sh)

**Как это работает:**
```
Ваш компьютер
     ↓
  Русский прокси #1
     ↓
  Иностранный прокси #2
     ↓
Целевой сервер
```

**Что добавляет stargate-chain.sh:**
- 🔁 Автоматически создаёт дубликаты всех подписок с суффиксом `-via-ru`
- 🔗 Настраивает `dialer-proxy: RU_AUTO` для маршрутизации через российские серверы
- 🚫 Исключает российские прокси из цепочек (`exclude-filter: *exclude_ru`) чтобы избежать петель
- 🚫 Исключает WireGuard из цепочек (`exclude-type: wireguard`) — они не поддерживают двойную маршрутизацию
- 📦 Сохранает поддержку AWG, SRV и VETH провайдеров из основного entrypoint.sh

**Особенности:**
- 🔗 Двойная маршрутизация для восстановления доступности интернета
- 🚀 Требует нескольких живых RU прокси в подписках
- ⏱️ Пинг (задержка) будет больше — два прокси вместо одного
- 💾 Память: ~50-90 МБ в зависимости от подписок (удвоенное количество провайдеров)
- 🔄 Создаёт дополнительные группы `VIA_RU_AUTO`, `VIA_RU_FASTEST` для автоматической маршрутизации через РФ

**⚠️ Важно знать:**
- ⏱️ Из-за двойной маршрутизации пинг выше — онлайн-игры могут быть медленнее
- 📁 **Требует загрузки ДВУХ файлов**, а не одного
- 🔗 Работает только с подписками (`SUB1`, `SUB2` и т.д.) — без них не сработает, но можете попытаться изменить его под свои нужды.
- 🔄 Удваивает количество провайдеров в конфигурации (оригинал + цепочка)

**Установка цепочек:**

1. Загрузите оба файла:
   - [stargate-chain.yaml](chain/stargate-chain.yaml) → в `/usb1/docker_configs/mihomo_mikrotik/template`
   - [stargate-chain.sh](chain/user_sh/stargate-chain.sh) → в `/usb1/docker/mihomo_mikrotik/user_sh`

2. Перезапустите контейнер для активации скрипта

3. **Обязательно включите "Auto disconnect idle UDP"** в zashboard или подобной панели, чтобы неактивные или подвисшие соединения при переключении цепочки закрывались (например, каждые 10 минут)

**Требования:**
- Несколько рабочих RU прокси в подписках (для надежности)
- Закрытие неактивных UDP-соединений
- Достаточно ли у роутера памяти для удвоения подписок?

## Управление маршрутизацией: переключатели

**Что это такое?** Переключатели (switches) — это как кнопки на панели управления. Они помогают выбрать, через какой именно прокси отправить трафик того или иного сервиса:
- Для YouTube могу выбрать один прокси
- Для Telegram — другой
- Для игр — третий

Некоторые переключатели вы настраиваете вручную, другие работают автоматически.

### Переменные шаблонов

Шаблоны используют переменные окружения, которые подставляются при запуске контейнера. Вот основные из них:

**Обязательные переменные:**

| Переменная                    | Описание                      | Пример значения                               |
|-------------------------------|-------------------------------|-----------------------------------------------|
| `LOG_LEVEL`                   | Уровень логирования           | `info`, `debug`, `warning`, `error`, `silent` |
| `IPV6`                        | Включение IPv6                | `true` / `false`                              |
| `EXTERNAL_CONTROLLER_ADDRESS` | Адрес API контроллера         | `0.0.0.0`, `127.0.0.1`                        |
| `UI_PORT`                     | Порт веб-интерфейса           | `9090`                                        |
| `EXTERNAL_UI_PATH`            | Путь к UI файлам              | `ui`, `metacubexd`                            |
| `EXTERNAL_UI_URL`             | URL для загрузки UI           | `https://github.com/...`                      |
| `UI_SECRET`                   | Секрет API                    | произвольная строка                           |
| `DNS_ENABLE`                  | Включение DNS                 | `true` / `false`                              |
| `DNS_USE_SYSTEM_HOSTS`        | Использование системных hosts | `true` / `false`                              |
| `MIXED_PORT`                  | Порт mixed прокси             | `1080`                                        |

**Переменные подписок:**

| Переменная                     | Описание                           | Значения по-умолчанию                  |
|--------------------------------|------------------------------------|----------------------------------------|
| `SUB1`, `SUB2`, ...            | URL подписок на прокси-серверы     |                                        |
| `SRV0`, `SRV1`, ...            | Локальные серверы в формате YAML   |                                        |
| `HEALTH_CHECK_URL`             | URL для проверки прокси            | `https://www.gstatic.com/generate_204` |
| `HEALTH_CHECK_EXPECTED_STATUS` | Ожидаемый HTTP статус              | `204`                                  |
| `PROVIDER_INTERVAL`            | Интервал обновления подписок (сек) | `3600`                                 |

**Переменные TUN/TPROXY (генерируются автоматически):**

| Переменная                  | Описание                                                  | Варианты                    |
|-----------------------------|-----------------------------------------------------------|-----------------------------|
| `TUN_IN_AUTOCONFIG`         | Блок конфигурации TUN/TPROXY (генерируется entrypoint.sh) |                             |
| `TUN_STACK`                 | Стек TUN                                                  | `system`, `mixed`, `gvisor` |
| `TUN_AUTO_DETECT_INTERFACE` | Автоопределение интерфейса                                | `true` / `false`            |
| `TUN_AUTO_ROUTE`            | Автоматическая маршрутизация                              | `true` / `false`            |
| `TUN_AUTO_REDIRECT`         | Автоматический редирект                                   | `true` / `false`            |
| `TUN_INET4_ADDRESS`         | Адрес TUN интерфейса                                      | `172.16.0.1/30`             |

**Автоматически генерируемые:**

| Переменная              | Описание                                                                      |
|-------------------------|-------------------------------------------------------------------------------|
| `HWID`                  | Hardware ID (генерируется из UUID) для подписок с ограничением по устройствам |
| `PROVIDERS_BLOCK`       | Блок подписок на прокси (генерируется из `SUB*`, `SRV*`, AWG `.conf` файлов)  |
| `PROVIDERS_LIST`        | Список провайдеров для групп (генерируется автоматически)                     |
| `PROVIDERS_CHAIN_BLOCK` | Блок цепочечных провайдеров (только для Chain шаблона)                        |
| `PROVIDERS_CHAIN_LIST`  | Список цепочечных провайдеров (только для Chain шаблона)                      |

### Ручные переключатели 👆

Эти вам нужно настраивать самостоятельно в панели управления. **При первом запуске обязательно выберите прокси вручную:**

| Переключатель | Назначение                                    | Что это значит                                       |
|---------------|-----------------------------------------------|------------------------------------------------------|
| `SELECTOR`    | **Главный** для большинства сервисов          | Для трафика, не распределеяемого по сервисам         |
| `MANUAL`      | Дополнительный выбор иностранного прокси      | Если SELECTOR не подходит — выберите вручную здесь   |
| `RU_MANUAL`   | Дополнительный выбор российского прокси       | То же, но специально для RU-прокси                   |
| `AWG_MANUAL`  | Специальные прокси (WireGuard, WARP, Amnezia) | Для сервисов, которым нужны эти безопасные протоколы |

### Автоматические переключатели ⚙️

Эти работают сами по себе. **Никогда их не трогайте в панели!** Если вы их измените, автоматика сломается.

**Типы Fallback** — они выбирают прокси автоматически:
- `AUTO` — сам выбирает лучший иностранный прокси
- `RU_AUTO` — сам выбирает рабочий российский прокси
- `AWG_AUTO` — сам выбирает безопасный WireGuard/WARP прокси
- `VIA_RU_AUTO` — сам проводит трафик через РФ в заграницу **(только Chain шаблон)**
- `FAILOVER` / `RU_FAILOVER` — если основной прокси упал, автоматически переключаются на резервный

**Типы URLTest** — они выбирают НА ОСНОВЕ СКОРОСТИ:
- `FASTEST` — постоянно проверяет и использует САМЫЙ БЫСТРЫЙ иностранный прокси
- `RU_FASTEST` — САМЫЙ БЫСТРЫЙ российский прокси
- `AWG_FASTEST` — САМЫЙ БЫСТРЫЙ безопасный прокси
- `VIA_RU_FASTEST` — проверяет скорость и выбирает быстрейший путь через РФ в заграницу **(только Chain шаблон)**

### 🆘 Сломалась автоматика?

**Не волнуйтесь, это легко исправить:**

1. Откройте панель управления проксями
2. Найдите сбитый переключатель (около него отображается иконка замочка)
3. Нажмите на него **правой кнопкой мыши**
4. или Нажмите кнопку спидометра

Тогда автоматика восстановится и всё будет работать снова.

### Технические параметры (для любопытных)

**Tolerance в URLTest:** Это время отклика в миллисекундах. 
- Текущее: 50 мс
- Измените на 10 мс если хотите, чтобы прокси переключались более агрессивно (чаще)
- Оставьте 80-100 мс если хотите большей стабильности

**Фильтрация РФ прокси:** Система ищет:
- Флаг 🇷🇺 в названии
- Или текст "RU"

Если в ваших прокси используются другие обозначения (вместо "RU"), отредактируйте фильтр `filter` в шаблонах

**WireGuard/WARP прокси:** Они не могут работать в цепочках (double proxy) из-за особенностей маршрутизации внутренних адресов, разным тоннелям важно знать про маршруты друг друга, поэтому автоматически исключаются из режима Chain. Но вы всё равно можете их использовать через `AWG_*` переключатели в обычных шаблонах (Lite, Full, Mini)

**💡 Совет по свободной памяти роутера:** Если оставить панель управления открытой в браузере — она будет избыточно грузить память и ресурсы оборудования. Если закроете браузер или вкладку когда ничего не настраиваете — сэкономите ~10 МБ ОЗУ. Не все агрессивные дашборды полезны.

## Оптимизация подписок

### Очистка подписок (рекомендуется)

Если вы любите бесплатные подписки из интернета, они часто содержат рекламу, бесполезные или дырявые конфигурации, которые раскрывают заинтересованным лицам ваши методы доступа в интернет.

Используйте [фильтр подписок](https://github.com/Viktor45/sub-filter) для удаления:
- неработающих серверов
- опасных конфигураций
- мусорных данных

Если ваши подписки уже чистые (платные), фильтр необязателен.

### Количество подписок

**Совет:** Используйте несколько разных подписок через `SUB1`, `SUB2`, `SUB3`:
- Повышает надежность (если одна подписка не загрузится)
- Больше вариантов для выбора прокси
- Критично для режима **Chain** (иначе может переключиться на прямое соединение по исчерпанию доступных серверов)

### Важные замечания

#### Кодировка

**Обязательно сохраняйте файлы шаблонов в UTF-8 без BOM:**
- Откройте конфиг в VS Code или другом редакторе
- Убедитесь в нижнем правом углу: `UTF-8` (не `UTF-8 with BOM`)
- BOM может помешать Mihomo распознать конфиг

#### Ограничения памяти

Значения потребления памяти основаны на экспериментах:
- Архитектура: 32-bit ARM
- Количество подписок: 1
- Прокси на подписку: ~30 штук

Ваши значения могут отличаться в зависимости от оборудования и количества прокси.
